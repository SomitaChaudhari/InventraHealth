## Cleaning

All five datasets were cleaned using Python (pandas, numpy) in a dedicated cleaning pipeline:

**`hospital_supply_chain_data_cleaning.ipynb`**

This notebook is not part of the main project repo but documents every transformation applied.
Refer to it to understand exactly what changed between raw and clean files.

The cleaning pipeline covered the following across all datasets:

- Column name standardisation — stripped whitespace, removed special characters, normalised casing
- Date parsing — all date and datetime columns coerced to consistent formats using `pd.to_datetime`
- Numeric coercion — removed `$` and `,` characters, converted strings to float/int, flagged non-parseable values
- Computed columns added — derived fields calculated from existing columns (see per-table notes below)
- Data quality flags added — boolean flag columns created to surface known issues without removing rows
- No rows were dropped — all original records preserved; issues were flagged, not deleted

A full issues log (`data_quality_log.csv`) was generated by the cleaning notebook documenting every issue found, the count, and the action taken.

## Per-table cleaning notes

### inventory_data_clean.csv
Original columns: Date, Item_ID, Item_Type, Item_Name, Current_Stock, Min_Required, Max_Capacity, Unit_Cost, Avg_Usage_Per_Day, Restock_Lead_Time, Vendor_ID

Cleaning applied:
- Date parsed to datetime
- All numeric columns (Current_Stock, Min_Required, Max_Capacity, Unit_Cost, Avg_Usage_Per_Day, Restock_Lead_Time) coerced from string, $ and commas removed
- Item_Type and Item_Name whitespace stripped and title-cased

Computed columns added:
- `Item_Key` — composite key (Item_ID + Item_Name) for stable joins
- `Days_Until_Stockout` — Current_Stock / Avg_Usage_Per_Day
- `Restock_Alert` — True when Days_Until_Stockout <= Restock_Lead_Time
- `Flag_OverStock` — True when Current_Stock > Max_Capacity
- `Flag_BadMinReq` — True when Min_Required appears suspiciously low (possible data entry error)

### financial_data_clean.csv
Original columns: Date, Expense_Category, Amount, Description

Cleaning applied:
- Date parsed to datetime
- Amount cleaned of $ and comma characters and coerced to numeric
- Negative amounts flagged for manual review (not removed)

Computed columns added:
- `Year_Month` — YYYY-MM aggregation key for monthly reporting
- `Item_Type_Mapped` — normalised item type derived from Expense_Category for cross-table joins with inventory

---

### patient_data_clean.csv
Original columns: Patient_ID, Admission_Date, Discharge_Date, Primary_Diagnosis, Procedure_Performed, Room_Type, Bed_Days, Supplies_Used, Equipment_Used, Staff_Needed

Known issue in source data: 195 records have Discharge_Date earlier than Admission_Date. Original dates were preserved as the source of truth. Bed_Days was recalculated as abs(Discharge_Date − Admission_Date) so the value is always positive and consistent.

Cleaning applied:
- Admission_Date and Discharge_Date parsed from datetime strings (both carried timestamps in raw data)
- Bed_Days overwritten with recalculated absolute day difference

Computed columns added:
- `Flag_InvertedDates` — True where Discharge_Date < Admission_Date (195 rows)
- `Admission_Date_Only` — date-only version of Admission_Date, safe to use for date arithmetic
- `Discharge_Date_Only` — date-only version of Discharge_Date, safe to use for date arithmetic

Note: Always use `Admission_Date_Only` and `Discharge_Date_Only` for any date calculations, not the raw datetime columns.

---

### staff_data_clean.csv
Original columns: Staff_ID, Staff_Type, Shift_Date, Shift_Start_Time, Shift_End_Time, Current_Assignment, Hours_Worked, Patients_Assigned, Overtime_Hours

Known issues in source data: Shift_Date carried full datetime timestamps (should be date only). Shift_Start_Time and Shift_End_Time were 12-hour format strings (e.g. "06:00 PM").

Cleaning applied:
- Shift_Date stripped to date-only using `.dt.normalize()`
- Shift_Start_Time and Shift_End_Time parsed from 12-hour format strings

Computed columns added:
- `Shift_Start_Time_Parsed` — 24-hour parsed version of Shift_Start_Time, safe for time arithmetic
- `Shift_End_Time_Parsed` — 24-hour parsed version of Shift_End_Time, safe for time arithmetic
- `Shift_Duration_Hours` — computed duration from parsed start and end times
- `Flag_HoursExceedShift` — True when Hours_Worked > Shift_Duration_Hours (130 rows, 26%)
- `Expected_OT` — expected overtime based on scheduled duration vs standard 8-hour shift

---

### vendor_data_clean.csv
Original columns: Vendor_ID, Vendor_Name, Item_Supplied, Avg_Lead_Time (days), Cost_Per_Item, Last_Order_Date, Next_Delivery_Date

Cleaning applied:
- Column names standardised — parentheses and spaces replaced, trailing underscores removed
- Last_Order_Date and Next_Delivery_Date parsed to datetime
- Avg_Lead_Time_days and Cost_Per_Item coerced to numeric, $ removed
- Vendor_Name and Item_Supplied whitespace stripped and title-cased
- Logical check applied: Next_Delivery_Date before Last_Order_Date flagged

Computed columns added:
- `Lead_Time_Actual_Days` — actual days between Last_Order_Date and Next_Delivery_Date, used to detect vendor delays by comparing against Avg_Lead_Time_days
